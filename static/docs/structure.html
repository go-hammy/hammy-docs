Hammy Project Structure

The Hammy Project is organized into several key components, each with distinct roles:

  • serverPlugin: This is the main plugin of the Hammy server. It starts the server components and is responsible for processing HTTP requests, delivering files, and overseeing server functions. The serverPlugin directs requests, runs PHP scripts, and delivers HTML files. It also calls other functions like htaccessFunction and cacheFunction to enhance its capabilities.
  • cacheFunction: A function that is part of the serverPlugin, enhancing performance by storing and retrieving cached responses. It reduces the need to access files from the disk repeatedly by serving stored responses for identical requests.
  • htaccessFunction: Another function called by the serverPlugin, responsible for managing and applying .htaccess rules to incoming HTTP requests, ensuring proper access control and URL rewriting.
  • bannerPlugin: A separate plugin that displays the Hammy banner, providing a visual representation of the server's identity and status.
  • content: This folder is designed for use with Docker or Docker Compose. It holds the static and dynamic content that the server will deliver. In a Docker setup, this directory is mapped to /var/www/html within the container.
  • serverPlugin/pages: This directory includes custom error pages such as hammy-404.html and hammy-500.html, which are displayed when the server encounters 404 or 500 errors, respectively. These pages are used when there is no 500.html or PHP file in the default /var/www/html.
